WatchTurm Control Room

Roadmap & Delivery History - Release Management dashboard for multi-platform environments

Phase 1 - Foundation & Visual Control Room

DONE
  • Lightweight SPA architecture (HTML + CSS + JS)
  • Unified left navigation: Overview / Platforms / Tools / Admin
  • Overview dashboard with pipeline radar (DEV / QA / UAT / PROD)
  • Visual project slices with instant environment visibility
  • Glass UI design system (dark & light theme, cards, spacing, typography)
  • JSON-driven data model powered by real snapshot

Phase 2 - Interaction, State & Release Reality

DONE
  • Clickable environment tiles per project
  • Inline expandable environment quick links (CMS, FE/brands, ArgoCD, infra)
  • Environment details: component table with tags, branches, builds, deployers
  • Compare mode: select multiple environments, see mismatched versions
  • Right sidebar with documentation and alerts
  • Release History with per-project tables and tag-change events
  • Ticket Tracker with multi-PR scenarios and environment presence
  • Clear separation between Platforms (project views) and global Tools

Phase 3 - Real-time Integrations & Snapshot Engine

DONE
  • GitHub integration: PRs, merges, branches, commit history
  • TeamCity integration: build metadata, triggered-by, branch info
  • ArgoCD integration: health & sync status per environment
  • Jira integration: ticket status, summary, fix versions
  • Datadog integration: observability metrics, live proxy, health signals
  • Snapshot scheduler: automated background refresh with ETA tracking
  • Release History: append-only enterprise storage with 60-day backfill
  • Configuration wizard: admin panel for tenant setup and API tokens

Phase 4 - Runbooks & Operational Tools

IN PROGRESS
  • Scope Checker - compare baseline branch vs default branch: tickets, commits, PRs
  • Drift Checker - detect changes in release branch not yet in default branch
  • Release Diff - compare two releases (e.g. 0.20.0 vs 0.21.0): added tickets & commits per repo
  • Strikethrough for repos with missing branches (visual feedback)
  • Clickable commits URL to view changes on GitHub
  • Ticket prefix filtering (case-insensitive, comma-separated)
  • Input hints for all runbook configuration fields
  • More runbooks in development (readiness checks, promotion validators)
Runbooks are actively developed. New runbooks will be added based on team feedback.

Phase 5 - Statistics & Reminders

DONE
  • Statistics dashboard: deployment frequency, lead time, top deployers
  • Per-project and per-environment breakdown
  • Time window selector (7d / 30d / 90d)
  • Reminders & Signals panel: stale deploy warnings, QA vs UAT diff hints
  • Deployment-focused alerts (no Datadog noise in reminders)

Phase 6 - UI Polish & Enterprise Ready

DONE
  • Dark & Light theme with consistent styling
  • Unified dropdowns, inputs, and buttons across the app
  • Navigation dividers for better visual hierarchy
  • Overview layout: balanced panels (radar, history, integrations)
  • Generic naming (no hardcoded company names in UI)
  • Responsive design for various screen sizes

Phase 7 - Future Plans

PLANNED
  • Advanced runbooks: promotion validators, rollback checkers
  • Cross-project comparisons and benchmarks
  • Historical analytics and trend detection
  • Optional data export (CSV / JSON)
  • Webhook notifications for critical events
  • Role-based access (read-only by design, optional write actions)
Future plans are indicative and may change based on team needs.
